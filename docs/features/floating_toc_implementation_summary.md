## 技术亮点

### 1. 高性能滚动监听

**使用 IntersectionObserver 替代传统 scroll 事件**：
- ✅ 异步执行，不阻塞主线程
- ✅ 自动管理，无需手动清理
- ✅ 性能开销小，可同时观察多个元素

### 2. 弹性动画效果

**使用 cubic-bezier 缓动函数**：
```css
transition: width 0.3s cubic-bezier(0.34, 1.56, 0.64, 1);
```
- ✅ 提供弹性效果，增强用户体验
- ✅ GPU 加速，保证 60fps 流畅度

### 3. 响应式设计

**PC 和移动端完全不同的交互模式**：
- PC：悬停展开，不占用屏幕空间
- 移动：按需弹出，优化触摸操作

### 4. 主题自适应

**使用 CSS 变量实现主题系统**：
- ✅ 自动跟随页面主题切换
- ✅ 易于扩展和定制

### 5. 代码质量

**遵循最佳实践**：
- ✅ 使用 'use strict' 严格模式
- ✅ 详细的代码注释（66+ 处）
- ✅ 完善的错误处理
- ✅ 模块化设计，易于维护

## 使用指南

### 快速开始

1. **启动 API 服务器**：
   ```bash
   python main.py --start
   ```

2. **访问转录结果页面**：
   - TOC 会自动显示在页面右侧
   - 如果没有标题，TOC 会自动隐藏

3. **独立测试页面**：
   - 直接在浏览器打开：`tests/features/test_floating_toc.html`
   - 或通过 HTTP 服务器访问

### 自定义配置

编辑 `floating-toc.js` 中的 `CONFIG` 对象：

```javascript
const CONFIG = {
    // 调整 IntersectionObserver 触发阈值
    OBSERVER_OPTIONS: {
        threshold: 0.5,
        rootMargin: '-100px 0px -60% 0px'
    },

    // 调整移动端断点
    MOBILE_BREAKPOINT: 768
};
```

## 常见问题快速解决

### Q: TOC 没有显示？

**检查清单**：
1. 页面是否有"内容总结"区块？
2. "内容总结"内是否有 H1-H4 标题？
3. 浏览器控制台是否有错误？
4. JS/CSS 文件是否成功加载？

### Q: 如何调整 TOC 位置？

修改 `floating-toc.css`：
```css
.floating-toc-container {
    top: 100px;    /* 顶部距离 */
    right: 20px;   /* 右侧距离 */
}
```

### Q: 如何更改主题颜色？

修改 CSS 变量：
```css
:root {
    --toc-active: #your-color;        /* 高亮颜色 */
    --toc-indicator-start: #color1;   /* 渐变起始 */
    --toc-indicator-end: #color2;     /* 渐变结束 */
}
```
